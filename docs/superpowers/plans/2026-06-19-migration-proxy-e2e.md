# Migration proxy dev-server → cluster e2e Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an e2e suite that brings up a source `TemporalDevServer` and a Postgres-backed target `TemporalCluster`, provisions the `TemporalMigration` proxy, and proves that a workflow started through the proxy runs on the **source** before cutover and on the **target** after cutover.

**Architecture:** A lightweight Go runner (separate module) provides a `worker` subcommand (runs an `EchoWorkflow` that returns the executing worker's cluster label via a `WhoAmIActivity`) and a `client` subcommand (starts the workflow through the proxy and asserts which cluster ran it). Workers connect **directly** to each cluster (the proxy always routes `Poll` to source); only the client goes through the proxy. A chainsaw suite wires it together; CI runs it nightly + on dispatch.

**Tech Stack:** Go + `go.temporal.io/sdk`, Kubernetes, chainsaw, kind, CloudNativePG, Helm.

## Background facts (verified against the repo)

- **Source = address only; target must be a `TemporalCluster`.** `targetRef.name` is resolved as `var cluster temporalv1alpha1.TemporalCluster` (`internal/controller/temporalmigration_controller.go:81-82`).
- **The controller dials `spec.source.address` from the operator pod (temporal-system)** for drain counting (`temporalmigration_controller.go:202-214`), so `source.address` MUST be a namespace-qualified FQDN.
- **Proxy routing** (`internal/proxy/director.go:48-63`): passthrough → all to source; cutover → `Start`=target, `Existing`=target/source-fallback, `Poll`=source. Therefore **workers must connect directly** to each frontend.
- **Service names:** dev server frontend = `<name>-devserver` (`internal/resources/devserver.go:44`); cluster frontend = `<name>-frontend` (`internal/resources/service.go:32`); proxy service = `<migration-name>-proxy` (`internal/resources/migrationproxy.go:35`). All listen on `:7233`.
- **`status.proxyEndpoint`** = `<migration-name>-proxy.<ns>.svc.cluster.local:7233` (`temporalmigration_controller.go:99`); phases are `Pending|Passthrough|Cutover|Draining|Complete` (`api/v1alpha1/temporalmigration_types.go:139-143`).
- **The `migration` namespace must exist on both clusters.** The registered Temporal namespace name equals the `TemporalNamespace` object's `metadata.name` (`internal/controller/temporalnamespace_controller.go:193` — `Name: ns.Name`), so two objects can't both register `migration` in one k8s namespace. Instead: create it on the **source** via `TemporalDevServer.spec.namespaces: ["migration"]` (`internal/resources/devserver.go:115-117`, `api/v1alpha1/temporaldevserver_types.go:46-50`), and register it on the **target** via a single `TemporalNamespace` object named `migration` with `clusterRef.kind: TemporalCluster` (`test/e2e/devserver/02-namespace.yaml`, `api/v1alpha1/temporalnamespace_types.go:25-32`).
- **CNPG fixtures to reuse:** `test/e2e/postgres/01-fixtures-cnpg.yaml` (CNPG `Cluster` named `temporal-pg`, app secret `temporal-pg-app`) and `test/e2e/postgres/02-secrets.yaml` (the `create-visibility-db` Job).
- **e2e infra:** the CI workflow already installs CNPG + cert-manager for every suite, builds/loads `temporal-operator:e2e`, installs the operator via Helm, side-loads Temporal images by scanning the suite dir, then runs `./bin/chainsaw test --test-dir test/e2e/<suite> --config .chainsaw.yaml` (`.github/workflows/e2e.yml`).
- **Do NOT run `make helm-chart`** (issue #82). No product-code changes in this plan.

Replace the entire contents of `test/e2e/migration/runner/main.go` with:

```go
// Command migration-runner is a lightweight Temporal worker/client used by the
// migration-proxy e2e suite. The EchoWorkflow runs WhoAmIActivity, which each
// worker registers to return its own cluster label, so the client can prove
// which cluster executed a workflow.
package main

import (
	"context"
	"flag"
	"log"
	"os"
	"time"

	"go.temporal.io/sdk/activity"
	"go.temporal.io/sdk/client"
	"go.temporal.io/sdk/worker"
	"go.temporal.io/sdk/workflow"
)

// taskQueue is shared by the source worker, the target worker, and the client.
const taskQueue = "migration-e2e"

// EchoWorkflow executes WhoAmIActivity and returns the executing worker's
// cluster label.
func EchoWorkflow(ctx workflow.Context) (string, error) {
	ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
		StartToCloseTimeout: 30 * time.Second,
	})
	var who string
	if err := workflow.ExecuteActivity(ctx, "WhoAmIActivity").Get(ctx, &who); err != nil {
		return "", err
	}
	return who, nil
}

func main() {
	if len(os.Args) < 2 {
		log.Fatal("usage: migration-runner <worker|client> [flags]")
	}
	switch os.Args[1] {
	case "worker":
		runWorker(os.Args[2:])
	case "client":
		runClient(os.Args[2:])
	default:
		log.Fatalf("unknown subcommand %q (want worker|client)", os.Args[1])
	}
}

// runWorker connects directly to a cluster frontend and serves EchoWorkflow,
// registering a WhoAmIActivity that returns this worker's cluster label.
func runWorker(args []string) {
	fs := flag.NewFlagSet("worker", flag.ExitOnError)
	address := fs.String("address", "", "frontend address host:port")
	namespace := fs.String("namespace", "migration", "Temporal namespace")
	cluster := fs.String("cluster", "", "cluster label this worker reports")
	_ = fs.Parse(args)
	if *address == "" || *cluster == "" {
		log.Fatal("worker: --address and --cluster are required")
	}

	c := mustDial(*address, *namespace)
	defer c.Close()

	w := worker.New(c, taskQueue, worker.Options{})
	w.RegisterWorkflow(EchoWorkflow)
	label := *cluster
	w.RegisterActivityWithOptions(
		func(ctx context.Context) (string, error) { return label, nil },
		activity.RegisterOptions{Name: "WhoAmIActivity"},
	)
	log.Printf("worker starting: cluster=%s address=%s namespace=%s", *cluster, *address, *namespace)
	if err := w.Run(worker.InterruptCh()); err != nil {
		log.Fatalf("worker exited: %v", err)
	}
}

// runClient starts EchoWorkflow through the proxy endpoint, waits for the
// result, and asserts it matches --expect. It exits non-zero on mismatch or
// timeout so a Kubernetes Job records the outcome.
func runClient(args []string) {
	fs := flag.NewFlagSet("client", flag.ExitOnError)
	address := fs.String("address", "", "proxy endpoint host:port")
	namespace := fs.String("namespace", "migration", "Temporal namespace")
	workflowID := fs.String("workflow-id", "", "workflow id")
	expect := fs.String("expect", "", "expected cluster label (source|target)")
	timeout := fs.Duration("timeout", 2*time.Minute, "overall timeout")
	_ = fs.Parse(args)
	if *address == "" || *workflowID == "" || *expect == "" {
		log.Fatal("client: --address, --workflow-id and --expect are required")
	}

	c := mustDial(*address, *namespace)
	defer c.Close()

	ctx, cancel := context.WithTimeout(context.Background(), *timeout)
	defer cancel()

	run, err := c.ExecuteWorkflow(ctx, client.StartWorkflowOptions{
		ID:        *workflowID,
		TaskQueue: taskQueue,
	}, EchoWorkflow)
	if err != nil {
		log.Fatalf("start workflow: %v", err)
	}
	log.Printf("started workflow id=%s run=%s", run.GetID(), run.GetRunID())

	var got string
	if err := run.Get(ctx, &got); err != nil {
		log.Fatalf("get result: %v", err)
	}
	if got != *expect {
		log.Fatalf("FAIL: workflow ran on %q, expected %q", got, *expect)
	}
	log.Printf("OK: workflow ran on %q as expected", got)
}

// mustDial retries dialing the frontend until success or a deadline, tolerating
// frontends that are still warming up when the pod starts.
func mustDial(address, namespace string) client.Client {
	deadline := time.Now().Add(3 * time.Minute)
	for {
		c, err := client.Dial(client.Options{HostPort: address, Namespace: namespace})
		if err == nil {
			return c
		}
		if time.Now().After(deadline) {
			log.Fatalf("dial %s (namespace %s): %v", address, namespace, err)
		}
		log.Printf("dial %s failed, retrying: %v", address, err)
		time.Sleep(3 * time.Second)
	}
}
```

# Postgres-backed target TemporalCluster via the upgrade proxy. Proves the
# routing transition: passthrough runs new workflows on the source; after

            file: 02-assert.yaml

        - assert:

            apiVersion: batch/v1

        - podLogs:
            selector: app=source-worker
        - podLogs:
            selector: job-name=client-passthrough
    - name: cutover
      try:
        - script:
            content: |
              kubectl patch temporalmigration orders-migration -n $NAMESPACE \
                --type merge -p '{"spec":{"cutover":true}}'
        # cutoverTime is set in the same reconcile that re-renders the proxy
        # Deployment into cutover mode, so waiting for it guarantees the new
        # proxy config has been applied before we check the rollout. (The
        # controller never sets phase "Cutover" — it goes Passthrough →
        # Draining → Complete — so do NOT assert phase here.)
        - assert:
            resource:
              apiVersion: temporal.bmor10.com/v1alpha1
              kind: TemporalMigration
              metadata:
                name: orders-migration
              status:
                (cutoverTime != null): true
        # Wait for the cutover-mode proxy pod to fully roll out before starting
        # the cutover client, so requests cannot hit the old passthrough pod.
        - script:
            content: |
              kubectl rollout status deployment/orders-migration-proxy -n $NAMESPACE --timeout=120s
    - name: assert-cutover-runs-on-target
      try:
        - apply:
            file: 07-client-cutover.yaml
        - assert:
            file: 07-assert.yaml

        - describe:
            apiVersion: batch/v1
            kind: Job
        - podLogs:
            selector: app=target-worker
        - podLogs:
            selector: job-name=client-cutover
        - podLogs:
            selector: app.kubernetes.io/instance=orders-migration
