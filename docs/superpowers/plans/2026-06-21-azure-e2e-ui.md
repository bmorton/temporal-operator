# Azure Web UI E2E Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enable the Temporal Web UI on the Azure standing cluster, have the Azure Chainsaw suite confirm it, and document how to reach it.

**Architecture:** Set `spec.ui.enabled: true` on the existing standing-cluster manifest so the operator creates an `azure-e2e-ui` Deployment + Service; verify it declaratively in the existing Chainsaw assert; document a `port-forward` access snippet in the docs site. No operator code changes — the UI feature already exists and connects to the frontend over gRPC (no DB connections).

**Tech Stack:** Kubernetes manifests, Chainsaw (kyverno-json JMESPath asserts), Hugo docs, Azure (`make azure-e2e-*`).

## Global Constraints

- Branch `test/azure-e2e-ui` is based on `main` (PR #88 already merged).
- Conventional Commits, every commit signed off (`git commit -s`), DCO enforced.
- Cluster/resource names: cluster is `azure-e2e`; the UI Deployment and Service are both named `azure-e2e-ui` (`UIName(clusterName)` = `<clusterName>-ui`); UI container port is `8080`, Service port name `http`.
- The UI talks to the frontend over gRPC — it adds **zero** Postgres connections, so the B1ms connection budget in `test/e2e/azure/README.md` is unaffected. Do not change `maxConns`.
- Do not add an ingress (the Azure e2e installs no ingress controller).
- The access snippet goes in `docs/content/installation/azure.md`, NOT in `test/e2e/azure/README.md`.

- [ ] **Step 1: Add the port-forward subsection**

In `docs/content/installation/azure.md`, immediately after the AGIC section (after the line that ends the `examples/cluster-azure-aks-ingress` reference and before `## Passwordless auth with Microsoft Entra + Workload Identity`), insert:

```markdown
## Accessing the UI without an ingress

When the UI is enabled (`ui.enabled: true`) but no ingress is configured — for
example the standing cluster from `make azure-e2e-deploy` — reach it with a
port-forward to the operator-generated `<cluster>-ui` Service (ClusterIP, port
8080):

```sh
kubectl -n azure-e2e port-forward svc/azure-e2e-ui 8080:8080
# open http://localhost:8080
```

Substitute your namespace and `<cluster>-ui` Service name for other clusters.
```
